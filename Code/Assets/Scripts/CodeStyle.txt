Рекомендуется именовать переменные следующим образом:
*имена переменных желательно должны содержательно описывать суть хранимых данных, и состоять из нескольких слов

*UpperCamelCase для наименования классов и методов.
*lowerCamelCase для наименования экземпляров классов.

*Имена классов - интерфейсов(соглашений) следует начинать с буквы I, пример "ISample"
*Имена StrangeIOC компонентов Mediator, Event, Root, Context - должны содержать соответствующее слово в названии
